# 任务类型定义

## Frontend（前端）

**说明**: UI 组件、页面、交互逻辑

**示例**:
- 实现登录页面
- 添加按钮组件
- 创建用户设置表单
- 实现数据可视化图表

**特征**:
- 输出可立即在浏览器中验证
- 依赖后端 API 或 mock 数据
- 需要考虑响应式设计

---

## Backend（后端）

**说明**: API、服务、业务逻辑、数据层

**示例**:
- 创建用户服务
- 实现 CRUD API
- 添加认证中间件
- 实现数据验证逻辑

**特征**:
- 需要定义 API 接口
- 可能需要数据库 schema 变更
- 需要考虑错误处理

---

## Database（数据库）

**说明**: Schema 设计、迁移、查询优化

**示例**:
- 设计用户表
- 添加索引优化查询
- 编写复杂 SQL 查询
- 数据迁移脚本

**特征**:
- 影响数据存储结构
- 需要考虑性能和扩展性
- 可能影响 API 设计

---

## Infra（基础设施）

**说明**: 配置、部署、CI/CD、容器化

**示例**:
- 设置 Docker 容器
- 配置 GitHub Actions
- 设置 Nginx 反向代理
- 配置环境变量

**特征**:
- 影响开发和部署流程
- 需要考虑安全性
- 通常作为其他任务的依赖

---

## Docs（文档）

**说明**: 文档、注释、README、API 文档

**示例**:
- 编写 API 文档
- 更新 README
- 添加代码注释
- 创建架构图

**特征**:
- 可以与其他任务并行
- 不影响功能实现
- 提高项目可维护性
